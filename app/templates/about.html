Project Architecture

Title

Cross-Scanner MRI Segmentation using Unsupervised Domain Adaptation

Problem Statement

Tumor segmentation models trained on MRI scans from one scanner (vendor/field strength/protocol) often degrade sharply when applied to scans from a different scanner, because of intensity, contrast, and noise differences — a phenomenon called domain shift. Collecting pixel-level tumor labels for every scanner is expensive. This project uses Unsupervised Domain Adaptation (UDA) to train a segmentation model that generalizes to a new ("target") scanner using only unlabeled images from it, while labels are only required from the original ("source") scanner.

Approach: Domain-Adversarial Neural Network (DANN)

A single U-Net encoder-decoder is extended with a second head — a domain classifier — attached to the bottleneck features through a Gradient Reversal Layer (GRL). During training:

  • The segmentation head is trained (with Dice + BCE loss) only on labeled source-domain images.
  • The domain classifier is trained to distinguish source vs target features (standard cross-entropy).
  • The GRL flips the gradient sign flowing back from the domain classifier into the shared encoder, so the encoder is pushed to produce features the domain classifier cannot tell apart — i.e. domain-invariant features that transfer across scanners.
Tech Stack
  • Deep Learning: PyTorch (U-Net + GRL domain-adversarial training, DANN)
  • Classical ML metrics: Dice coefficient, IoU (Jaccard index) for evaluation
  • Backend: Flask (Python) — REST-ish routes for upload, inference, results, dashboard
  • Database: PostgreSQL (via SQLAlchemy) — falls back to SQLite for zero-config local runs
  • Frontend: Server-rendered HTML (Jinja2) + Bootstrap 5 + Chart.js for the evaluation dashboard
  • Image processing: OpenCV / Pillow / NumPy for preprocessing and overlay visualization
Dataset

Real cross-scanner brain-tumor MRI datasets (e.g. BraTS with multi-institution scanners) require licensed access and large downloads. To make the project runnable end-to-end without external data, a synthetic multi-scanner MRI generator (data/synthetic_mri.py) creates structurally realistic brain-slice images with tumor masks, then applies scanner-specific intensity transforms (gamma, bias field, noise) to simulate the domain gap between "Scanner A" (source, labeled) and "Scanner B" (target, unlabeled during training). The same pipeline works unchanged on real MRI slices — just point the training script at a real labeled dataset for the source domain and real unlabeled images for the target domain.

Evaluation

A baseline U-Net (source-only, no adaptation) and the UDA/DANN model are both trained and evaluated on held-out source and target domain data. The dashboard reports Dice score and IoU for both models on both domains, showing the Dice-score improvement that domain adaptation provides on the unseen target scanner.
